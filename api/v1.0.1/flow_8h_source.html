23/*
24 * A grain can be marked as invalid for multiple reasons. for example, an input application may have
25 * timed out before receiving a grain in time, etc. Writing grain marked as invalid is the proper way
26 * to make the ringbuffer <move forward> whilst letting consumers know that the grain is invalid. A consumer
27 * may choose to repeat the previous grain, insert silence, etc.
28 */
29#define MXL_GRAIN_FLAG_INVALID 0x00000001 // 1 << 0.
