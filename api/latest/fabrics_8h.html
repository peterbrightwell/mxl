◆ mxlFabricsEndpointAddress

typedef struct mxlFabricsEndpointAddress_t mxlFabricsEndpointAddress

Address of a logical network endpoint. This is analogous to a hostname and port number in classic ipv4 networking. The actual values for node and service vary between providers, but often an ip address as the node value and a port number as the service value are sufficient. node and service pointers are expected to live at least until the target or initiator setup function is executed and are internally cloned.

Configure the target. After the target has been configured, it is ready to receive transfers from an initiator. If additional connection setup is required by the underlying implementation it might not happen during the call to mxlFabricsTargetSetup, but be deferred until the first call to mxlFabricsTargetTryNewGrain().

  • "cqDepth" (number >= 1): the depth of the target's completion queue. Increase this for high-frame-rate or many-stream receivers, or when per-completion processing is slow, to avoid completion queue overflow. See the "Receiving grains" section of docs/Fabrics.md. When omitted, an implementation default is used.
out_infoAn mxlFabricsTargetInfo_t object which should be shared to a remote initiator which this target should receive data from. The object must be freed with mxlFabricsFreeTargetInfo().

The result code.
